ROOT CAUSE ANALYSIS:
═══════════════════

Your app is failing with these errors:
  ❌ Error loading messages: Missing or insufficient permissions
  ❌ Write at users/{uid} failed: Missing or insufficient permissions
  ⚠️ Could not determine other participant
  ❌ Listen for query at messages failed: Missing or insufficient permissions

These errors occur because your Firestore database has NO security rules set up.
Firestore defaults to DENY ALL access for security. You need to explicitly allow:
  • Reading message documents
  • Writing typing indicators
  • Reading user profiles for chat names
  • Reading/writing service applications
  • Reading reviews

WHAT THE RULES DO:
═══════════════════

/messages (in conversations/{id}/messages)
├─ ✅ READ: Any participant in the conversation can read all messages
├─ ✅ CREATE: Conversation participants can send messages
└─ ✅ UPDATE/DELETE: Users can only modify their own messages

/users/{uid}
├─ ✅ READ: Any authenticated user can read profiles (needed for chat names)
└─ ✅ WRITE: Users can only update their own profile

/conversations/{id}
├─ ✅ READ: Participants can read conversation details
├─ ✅ CREATE: Authenticated users can create conversations
└─ ✅ UPDATE/DELETE: Participants can modify their conversation

/conversations/{id}/typingIndicators/{userId}
├─ ✅ READ: Participants can see who's typing
└─ ✅ WRITE: Users can set their own typing status

/services/{id}
├─ ✅ READ: Authenticated users can browse services
└─ ✅ WRITE: Service providers can create/update their own services

/applications/{id}
├─ ✅ READ: Applicants and service providers can see applications
└─ ✅ CREATE: Users can apply for services
└─ ✅ UPDATE: Providers and applicants can modify status

/reviews/{id}
├─ ✅ READ: All authenticated users can read reviews
├─ ✅ CREATE: Users can submit reviews
└─ ✅ UPDATE/DELETE: Only reviewers can modify/remove their reviews

/transactions/{id}
├─ ✅ READ: Users can only read their own transactions
└─ ❌ UPDATE/DELETE: Disabled (immutable ledger for accounting)

TROUBLESHOOTING:
═════════════════

Q: "Still getting Permission denied errors"
A: 1. Check Firestore console shows your rules were published
   2. Make sure you copied the ENTIRE rules file (not partial)
   3. Clean build in Xcode and restart simulator
   4. Wait 2 minutes and try again (sometimes takes time to propagate)

Q: "Messages load but are empty"
A: 1. Try sending a message in the app
   2. Check Firestore console -> Collections -> messages
   3. If messages collection is empty, they weren't being saved
   4. This might indicate an issue with sendMessage() function

Q: "User names still showing 'User' or 'Provider'"
A: 1. Verify /users collection has user documents with displayName field
   2. Check that user IDs in conversations match actual users
   3. Try signing out and back in to refresh

Q: "Slow message loading (indexes needed)"
A: This is normal if indexes aren't built yet. Wait 5-10 minutes and try again.
   You can check index status in Firebase Console -> Indexes

Q: "Index creation failed or is stuck"
A: 1. Check Firebase Console -> Indexes
   2. If any are "BUILDING" for >10 minutes, try creating them again
   3. Verify the index fields are spelled exactly as specified
   4. Check that collection name is lowercase
